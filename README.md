# UrCheck

UrCheck — учебный проект для проверки договоров: загрузка `.docx`, извлечение текста, поиск релевантных рисков/норм и формирование структурированного ответа.

## Важно о текущем статусе (на 29 декабря 2025)

- Интеграция с OpenRouter в текущем виде (модуль `backend/services/llm_rag_service.py`) работает нестабильно и фактически не используется как рабочий путь.
- Из-за этого полный сценарий анализа через OpenRouter сейчас не гарантируется.
- API можно поднять локально, но финальный LLM-анализ зависит от внешнего провайдера и валидных ключей.

## Быстрый запуск (Windows / PowerShell)

1. Создать и активировать виртуальное окружение:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

2. Установить зависимости:

```powershell
python -m pip install --upgrade pip
pip install -r req.txt
```

3. (Опционально) задать ключ для режима через OpenAI:

```powershell
$env:OPENAI_API_KEY="your_api_key"
```

4. Запустить backend:

```powershell
cd backend
uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

5. Проверить, что сервер поднялся:

- `http://127.0.0.1:8000/health`
- `http://127.0.0.1:8000/docs`

## Что сейчас реально проверить

- Регистрация/логин и JWT-поток (`/auth/*`).
- Загрузка `.docx` и извлечение текста.
- Базовые пайплайны данных (парсинг, локальная подготовка JSON, работа с Chroma).

Ограничение: endpoint анализа зависит от внешней LLM и может падать при недоступности провайдера или ключа.

## Структура проекта

- `backend/` — FastAPI-приложение (роуты, сервисы, auth, модели, конфиг).
- `frontend/` — HTML-страницы интерфейса и статические файлы.
- `data/` — подготовленные данные, скрипты загрузки/поиска, локальные Chroma-хранилища.
- `parsing/` — парсинг исходных правовых документов (`.docx`) и сбор артефактов.



- OpenRouter-интеграция зафиксирована как ограничение текущего этапа.
